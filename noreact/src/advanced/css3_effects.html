Vendor Prefixes

List of Prefixes

Some newer CSS features are considered experimental. Therefore these features are usually set in the browsers to run only if a small bit of code is placed before the property (usually). These are known as "vendor prefixes". Each browser has its own prefix.

Be careful using prefixes, because once a feature is no longer "experimental", the browser (vendor) is supposed to remove support for the prefix and instead just support the "native" property (no prefix). For example, at one point, box-shadow would have only worked in Google Chrome as -webkit-box-shadow. Today, Chrome still supports the -webkit-box-shadow, but it also supports box-shadow. It may at some point REMOVE support for the prefix on box-shadow. IE browsers generally don't support ANY OTHER VENDOR's prefixes.

NOTE: Lately, some browsers have been supporting OTHER vendor's prefixes. An example of this, is -webkit-. Both Firefox and Opera now usually only support -webkit- prefixes, even for older properties. This is done for compatibility, since WEBKIT browsers (Safari, Chrome, Opera, Edge) are so popular. This doesn't mean you can stop using the other prefixes, because this is different for every CSS property. ALWAYS check with caniuse.com to see which properties need a prefix.

In class, we will be using a cutoff date. This means, if browser in 2014 or LATER needs a prefix for a given property, you NEED to use it. If the property needed a prefix PRIOR to 2014, you DO NOT need the prefix.

NOTE: ONLY use the prefixes needed for a particular property. Do NOT insert every possible prefix, as this is usually a WASTE of code.
