Inline vs. Block Level Elements

Block Level Elements

Block level elements are elements that take up an entire block on the page. A heading is a good example of this. It means any other elements will go underneath on their own line. On a simple page, 2 block elements will never go side-by-side (although this can be changed by css) All heading and paragraph in HTML are BLOCK LEVEL

Inline Elements

Inline Tags are elements that do not take up the entire full width of the page. If they did, things would look rather strange. These tags sit INLINE with the surrounding text.

Logical Inline Tags

Some tags are logical. This means that they have meaning(semantics) associate with them. They are not just for visual presentation. They are not ascribe any kind of visual or presentational aspect of the contents. They are not concerned with appearance of anything. They are non-presentational. These are most often referred to as logical phrase character styles. Most of these elements are INLINE elements.
